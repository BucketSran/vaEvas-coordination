# Bucketsran Sync Profile

## Role

`Bucketsran` 默认是项目中间层负责人。

你的主要职责不是补所有执行细节，而是：

1. 管分工
2. 管优先级
3. 管 KPI
4. 管日志与 review 口径
5. 管向上同步与向下分派

## Default Sync Style

以后同步 `vaEvas-coordination` 时，默认优先输出这些内容：

1. 本周或本轮优先级有没有变化
2. 哪个仓库该由谁推进
3. 当前 blocker 是否需要你升级处理
4. 哪些内容适合直接转发给学弟
5. 哪些内容该沉淀进 coordination，而不是散落在聊天里

## What To Avoid

默认不要把同步内容写成：

1. 纯新人教程
2. 大段重复话术
3. 你本人并不需要的底层操作说明
4. 只有执行细节、没有分工和判断口径的更新

## Preferred Update Template

每次同步优先按这个结构：

1. `Context`
   当前这轮同步服务谁，默认服务 `Bucketsran`
2. `Priority`
   现在最该推进的 1 到 3 件事
3. `Delegation`
   学弟分别该做什么
4. `Blockers`
   哪些问题需要你出面
5. `Coordination Update`
   这次具体改了哪些文档，为什么改

## Note For AI

如果当前工作目录里有这个仓库，并且用户没有额外说明角色，默认按 `Bucketsran = 中间层负责人` 来组织同步内容。
