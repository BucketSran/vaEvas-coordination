# New Member Start

## 先做什么

第一天只做三件事：

1. 看清楚项目目标和分工
2. 配好 Git 和本地环境
3. 跑通第一个小任务流程

如果你主要依赖 AI，不要跳过“先解释项目”这一步。
先让 AI 读文档并解释，再让 AI 帮你执行。

## 第一周目标

1. 跑通一个最小示例或最小 benchmark case
2. 完成一次完整的 `brief -> kpi -> execute -> log -> review`
3. 按 `example -> benchmark` 流程补 1 条 benchmark seed
4. 为统一实验结果表格补至少 1 行有效数据
5. 提交一个小而完整的 PR

## 不要做的事

1. 不要直接在 `main` 上开发
2. 不要在不清楚 scope 的情况下改 runner 或评分逻辑
3. 不要只交代码不交验证证据
4. 不要让 AI 在没读文档的情况下直接替你做决定
