# New Member Start

## 先做什么

第一天只做三件事：

1. 看清楚项目目标和分工
2. 配好 Git 和本地环境
3. 跑通第一个小任务流程

## 第一周目标

1. 跑通一个最小示例或最小 benchmark case
2. 完成一次完整的 `brief -> kpi -> execute -> log -> review`
3. 提交一个小而完整的 PR

## 不要做的事

1. 不要直接在 `main` 上开发
2. 不要在不清楚 scope 的情况下改 runner 或评分逻辑
3. 不要只交代码不交验证证据
